# 模拟数据生成

## 基准测试的当前模拟数据
欢迎将这些数据用于其他目的。
- `users.json`: 用户数据库，包含用户的电子邮件、地址和订单信息
- `products.json`: 产品数据库，每个产品都有变体（例如尺寸、颜色）
- `orders.json`: 可操作的订单数据库


查看 `../tools` 以了解基于当前模拟数据的模拟API。


### 模拟数据生成经验

阅读我们的论文以了解有关每个数据库生成过程的更多信息。一般来说，它包括以下阶段：

1. 设计每个数据库的类型和模式。可以使用GPT进行头脑风暴，但必须由人工决定，因为这是其他一切的基础。
2. 对于每个模式，确定哪些部分可以通过编程生成，哪些部分需要GPT。例如：
    - 产品类型（衬衫、灯、笔）和用户名（Sara、John、Noah）需要GPT生成
    - 产品价格和发货日期可以通过代码生成
3. 使用GPT生成种子数据（名字、姓氏、地址、城市等），然后使用程序将它们与其他代码生成的数据组合。可以使用GPT帮助编写这部分代码，但我认为基于代码的数据库构建比基于GPT的数据库构建更可靠（例如，提供一些示例用户资料并要求GPT生成更多——存在多样性和可靠性问题）。
